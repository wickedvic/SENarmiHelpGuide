Core Banking System Integration

Narmi acts as an intelligent abstraction layer between users and core banking systems, providing a unified experience regardless of the underlying core.

Understanding Core Banking Systems

Critical Knowledge: Core banking systems are back-end systems of record that process daily banking transactions. Each core has unique structures and communication requirements, which Narmi handles through custom integrations.

Types of Core Banking Systems

Core Type Description Example Systems Key Characteristics
User-Based Information associated with a person who has accounts IBS (FIS) Person ‚Üí Account relationships
Account-Based Information associated directly with accounts/memberships Symitar (Jack Henry) Account = Membership model
Hybrid Combination of user and account-based approaches Various vendors Flexible data organization

Core Hosting Models

  • On-Premise: Hosted at the financial institution (FI needs to provide connectivity)
  • Core Provider Hosted: Hosted offsite by the core vendor
  • Third-Party Hosted: Managed by a third-party service provider

Platform Architecture Components

Integration Layer

Standardizes data processing across all core types, managing the "shared language" between cores and Narmi products.

API Gateway

Native REST APIs provide secure, scalable access to all platform functionality.

Processing Engine

Handles deposits, loans, credit processing, and interfaces with General Ledger systems.

Security Layer

Enterprise-grade security with encryption, authentication, and compliance management.
